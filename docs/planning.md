# 차오름 차BTI 기획 노트

## 핵심 컨셉

- 대제목: 차오름 차BTI
- 소제목: 이 차는 이제 제겁니다
- 목적: 문답형 테스트를 통해 사용자에게 어울리는 차를 추천하고, SNS에 공유하기 좋게 만든다.
- 저장/통계 집계: 하지 않는다.
- 재사용성: 특정 축제 문맥에만 묶이지 않도록 질문 문항은 일반 상황에서도 쓸 수 있게 작성한다.

## 기술 방향

- 정적 웹앱으로 구현한다.
- 현재 MVP: Vite + React + TypeScript
- 배포: GitHub Pages
- 데이터: JSON 또는 TypeScript 상수로 관리한다.
- 공유:
  - 카카오톡: Kakao JavaScript SDK 사용
    - 기본 템플릿: `Kakao.Share.sendDefault`
    - 커스텀 템플릿: `Kakao.Share.sendCustom`
    - `data/app-config.json`에 JavaScript 키와 템플릿 ID를 넣어 활성화
  - 일반 공유: Web Share API 사용
  - 인스타그램/스토리: 결과 페이지 첫 화면을 결과별 정적 이미지로 미리 저장해 두고 저장/공유
  - fallback: 링크 복사, 이미지 다운로드

## 테스트 구성

- 질문 수: 8개
- 선지 수: 각 4개
- 결과 수: 12개
- 결과 설명 톤: 적당한 캐릭터화
- 결과명 형식: 형용사 + 차 이름 + 타입
- 공유 이미지 문구 밀도: 결과명 + 2줄 설명 + 추천 포인트 2개

## 스코어링 원칙

선택지는 특정 차 하나만 올리지 않고, 그룹 점수와 개별 차 보정 점수를 함께 가진다.

```txt
선지 선택
→ 그룹 점수 누적
→ 해당 그룹에 속한 결과 차들이 점수 획득
→ 개별 차 보정 점수 누적
→ 최종 결과 산출
```

최종 계산:

```txt
차 점수 = 선택된 선지의 그룹 점수 중 해당 차가 가진 그룹의 합
        + 선택된 선지의 개별 차 보정 점수
```

동점 처리:

```txt
1. 개별 차 보정 점수가 더 높은 결과 우선
2. 그래도 동점이면 priority 값이 낮은 결과 우선
```

## 주요 데이터 파일

- `data/tea-results.json`: 결과 12종, 그룹, 문구, 추천 차/제품, 우선순위
- `data/questions.json`: 질문 8개, 선지, 그룹 점수, 개별 차 보정 점수
- `index.html`: Vite 앱 진입점
- `src/main.tsx`: 테스트 플로우와 화면 컴포넌트
- `src/kakao.ts`: 카카오톡 공유 SDK 로딩과 템플릿 전송
- `src/scoring.ts`: 결과 계산 로직
- `public/story-images/`: 결과별 정적 공유 이미지
- `src/storyImage.ts`: 결과별 정적 공유 이미지 경로와 다운로드 로딩
- `src/styles.css`: 모바일 우선 UI 스타일
